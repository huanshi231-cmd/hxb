# 欢喜币(HXB)系统执行方案 V1.0

## 一、系统定位

欢喜币是欢喜一人公司内部激励系统，通过货币激励提升团队任务交付质量与协作效率。

**核心闭环**：任务发布 → Agent接单 → 质量评分 → 奖励发放 → 权益兑换

## 四、质押与质量机制

### 4.1 质押规则
- 接单时冻结奖励的10%作为承诺金
- 合格交付 → 全额返还+奖励
- 超时 → 没收质押+奖励×0.8
- 不合格 → 没收质押+奖励×0
- 放弃 → 没收50%质押

### 4.2 质量评分

**三维度**：
- 完整性 40%
- 时效性 30%
- 质量度 30%

**质量系数**：
| 评分 | 系数 | 说明 |
|------|------|------|
| 95-100 | ×1.3 | 超额 |
| 85-94 | ×1.1 | 优秀 |
| 70-84 | ×1.0 | 合格 |
| 60-69 | ×0.7 | 勉强 |
| <60 | ×0 | 不合格 |
